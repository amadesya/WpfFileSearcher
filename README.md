# Утилита асинхронного поиска и фильтрации файлов (WPF)

Приложение представляет собой легковесный инструмент для сканирования файловой системы, поиска файлов по части имени и их последующей фильтрации по метаданным (размер, дата создания).

Проект разработан в рамках изучения базовых возможностей платформы .NET и работы с системным вводом-выводом (I/O).

## Функциональные возможности

* **Рекурсивный поиск:** Сканирование как целевой директории, так и всех вложенных папок.
* **Фильтрация по размеру:** Ограничение результатов поиска по минимальному и/или максимальному объему файла (с конвертацией Кб в байты).
* **Фильтрация по временным меткам:** Поиск файлов, созданных не ранее указанной пользователем даты.
* **Потоковый перебор:** Использование `Directory.EnumerateFiles` вместо полного чтения массива в память, что оптимизирует потребление ресурсов на больших директориях.

## Текущий статус проекта и технический долг

На данном этапе проект является **функциональным прототипом** (MVP). В архитектуре приложения присутствует ряд упрощений, которые планируется устранить в ходе следующих этапов рефакторинга:

1. **Архитектура UI:** Логика фильтрации и обработки событий на данный момент реализована в слое Code-Behind (`MainWindow.xaml.cs`). В планах — полный перенос бизнес-логики на паттерн MVVM (Model-View-ViewModel).
2. **Многопоточность:** Текущие операции поиска выполняются в основном потоке интерфейса. Для предотвращения блокировки UI при сканировании крупных дисков (например, `C:\`) планируется перевод методов на асинхронную модель `async/await` и вынос задач в `Task.Run`.
3. **Обработка исключений:** Требуется расширить обработку ошибок доступа (`UnauthorizedAccessException`) при рекурсивном обходе защищенных системных директорий Windows.
